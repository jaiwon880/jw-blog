# **1. 대체 어떻게 공유를 해야 잘 공유했다고 소문이 날까?**
<br />

*본격적으로 실무에 투입되면서 개발팀 뿐만 아니라 다른 팀과의 소통이 필수적이었다. 그래서 용어가 익숙하지 않아 풀어서 설명하거나 여러번 설명을 해야 했던 경험이 있었다.*
<br />

이 글에서는 **'어떻게 효율적으로 소통할 수 있을까?'** 에 대한 글을 작성해보려고 한다.
<br />

*우선 나는, 진행 상황을 공유할 때 내 입장에서는 모두 설명했다고 생각했지만, 추가적으로 질문이 들어왔기 때문에 소통을 위한 전략이 필요했다.
그들의 배경지식이 어디까지인지 모르니, 개발 용어의 경우 최대한 개념을 풀어서 설명하거나 실생활 예제를 들어 설명했었다.
최대한 풀어서 이해하기 쉽도록 설명하고 질문이 들어오면 질문에 대한 답변에 이어서 답을 하며 커뮤니케이션을 했다.*
<br />

**직접 커뮤니케이션 한 예를 들어 보겠다.**
<br /><br />

```markdown
[소셜 로그인 (네이버, 카카오, 구글)]
카카오, 네이버 연동 및 개발 완료
구글의 경우 인증을 회사 도메인, 서비스 도메인 두 도메인을 하나의 인증 절차로 같이 쓰고 있는데,
회사 도메인 인증 또한 마쳐야 하는 상태입니다.
구글 로그인 코드는 구현이 되었고, 심사 승인이 완료되면 프론트 연동작업 후 완료됩니다.

[런칭 이벤트 페이지]
마케팅/ 디자인 수정 요청 사항에 대응 중

[어드민 관련]
회원, 댓글 관리 진행 중

```
<br />

**이렇게 커뮤니케이션을 했지만, 내가 나에게 피드백을 하자면**
<br /><br />

```markdown
[소셜 로그인 (네이버, 카카오, 구글)]
카카오, 네이버 심사(연동) 및 개발 완료
구글 로그인의 경우 회사 도메인, 서비스 도메인 두 도메인을 하나의 인증 절차를 사용하고 있는데 
현재 서비스 도메인만 인증이 완료된 상태이기 때문에 회사 도메인 인증이 되어야 하는 상태
(*도메인 = 주소: ex)google.com)

구글 로그인의 경우 코드는 개발이 완료 되어있고, 심사 승인이 완료되면
프론트(팀원)과 연동하면 개발이 완료됨

완료 예상 소요 기간: 승인 기간(3-5일)+ 프론트 작업(1일) 약 6일

[런칭 이벤트 페이지]
마케팅(어떤 요청 사항인지)/ 디자인(어떤 요청 사항인지) 요청 사항 대응 중
완료 시점: ~


[어드민 관련]
회원 관리 (100%)
댓글 관리 (80%) 진행중
```

<br />
이렇게 피드백을 할 수 있을 것 같다.

# **2. 마무리**
<br />

*내 상황에 대한 정확한 인지를 하고 정리하고 공유하는 것이 핵심인 것 같다. 두 번째는 전문용어를 풀어서 설명하기, 마지막으로 구체적이고 명확한 정보를 제공하며 소통하는 것이 중요할 것 같다.*
